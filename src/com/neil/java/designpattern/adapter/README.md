## 适配器的原理

适配器模式：将一个类的接口转换成另一种接口，让原本接口不兼容的类可以兼容
```sequence
被适配者->适配器:被适配者是Turkey
适配器->目标:目标是Duck
```
**总结**
1. 被适配者对用户是透明的，用来看到的就是目标的外观，是解耦的。
2. 用户调用适配器转化出来的目标接口方法
3. 适配器再调用被适配者的相关的接口方法
4. 用户收到反馈，感觉只是和目标接口交互

## 对象适配器
重复一下适配器的原理：将一个类的接口（或者方法）转换成另一种接口（或者方法），让原本接口（或者方法）不兼容的类可以兼容
两个例子：插头的适配器和点烟器适配器
对于用户来说，用户看不到被适配器的对象


## 类适配器
类适配器：通过多重继承目标接口和被适配者类方法来实现适配

多重继承，其中继承的目标接口部分来达到适配的  目的，而继承被适配者类的部分达到通过调用被适配者类里的方法来实现目标接口的功能
下面的叙述是针对`TurkeyAdapter2`类的。
>适配器角色TurkeyAdapter2扩展了WildTurkey,同时又实现了目标(Duck)接口。由于WildTurkey没有不能有一个long distance fly()方法，而目标接口又要求这个方法，因此适配器角色Adapter实现了这个方法。
